Request Body and Response Formats

(1) Person format

{
"email": String
"firstName": String
"lastName": String
"phones": [Number, Number, ...]
"address": { "street": String, "additional info": String, "persons": [person, person, ...]}
"hobbies": [hobby, hobby, ...]
}

(1) Phone format

{
"description": String
"number": int
"person": Person (1)
}

Errors

(e) All errors are reported using this format (with the HTTP-status code matching the number)

{ status : statusCode, "msg": "Explains the problem" }

  • (e1) :{ status : 400, "msg": "No persons found with this hobby" }
  • (e2) :{ status : 400, "msg": "No persons found living in this city" }
  • (e3) :{ status : 400, "msg": "Field ‘xxx’ is required" } (for example, no name provided)
  • (e4) :{ status : 400, "msg": "Person you are trying to edit cannot be found" }
  • (e5) :{ status : 400, "msg": "No phones found for person with that name" }
